# ST Analyzer (диаграмма состояний + карты Карно)

Это статический сайт (HTML/CSS/JS), поэтому его проще всего запустить через **GitHub Pages**.

## Вариант 1 (самый простой): через Settings → Pages

1. Запушьте репозиторий на GitHub.
2. Откройте репозиторий → **Settings** → **Pages**.
3. В разделе **Build and deployment**:
   - **Source**: `Deploy from a branch`
   - **Branch**: `main` (или ваша основная ветка)
   - **Folder**: `/ (root)`
4. Нажмите **Save**.
5. Через 1–3 минуты сайт будет доступен по адресу вида:
   - `https://<ваш-логин>.github.io/<имя-репозитория>/`

## Вариант 2: автодеплой через GitHub Actions (уже добавлен)

В репозитории есть workflow `.github/workflows/pages.yml`, который деплоит сайт при пуше в `main`.

Что нужно сделать один раз:

1. Откройте **Settings** → **Pages**.
2. В **Source** выберите `GitHub Actions`.
3. Запушьте изменения в ветку `main`.
4. Перейдите во вкладку **Actions** и дождитесь успешного workflow `Deploy static site to GitHub Pages`.

После этого сайт будет доступен по GitHub Pages URL.

## Локальный запуск (перед публикацией)

Из корня проекта:

```bash
python3 -m http.server 8080
```

Откройте: `http://localhost:8080`

---

## Важно

- Если репозиторий **private**, GitHub Pages может быть недоступен на бесплатном тарифе (зависит от типа аккаунта и тарифа).
- Если сайт не открылся сразу, подождите пару минут и обновите страницу.
